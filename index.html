Why testing matters

Analogy? If you could stop time and know the acceleration and mass of every particle in space you could predict the future. So with software, time does not stop and you are dreaming to have this amazing control of predicting the need of users, the browsers they will have, the various bugs that will bump various library versions, how users abuse the features you have and how priorities will change and features maybe even whole company strategies may pivot. And that is why you need testing, to know at any point in time that what you wanted to work a particular way, still works that way. And the easiest way to make sure your tests are relatively simple to write and your code is easy to reason about is to write the tests first, have them drive out the code reqruied, have them "Design" the code you need Test Driven Design.
